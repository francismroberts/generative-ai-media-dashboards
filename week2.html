🧩 Activity Guide
›

Activity 1: Voice Chat 🎙️

What it asks: Have one or more spoken conversations with ChatGPT Voice Chat or Gemini Live and write a short reflection comparing spoken vs written AI interaction.

Best approach: Use two short conversations — one practical ("Help me plan Week 2"), one reflective ("Does voice AI make users more likely to anthropomorphize?"). Connect to anthropomorphism and sycophancy readings.

Reflection angles: Voice feels more immediate and social. It may increase anthropomorphism. It reduces verification friction. Written chat is better for source-checking and audit trails.

Minimum viable: One 10-minute conversation + 150-word reflection. Strong: Two conversations + explicit connection to anthropomorphism/sycophancy.

Activity 2: Gemini Prompting Challenge 🎨

What it asks: Use Gemini image generation to match a provided real image, iterating on objects, colors, style, lighting, and tone. Keep full process history. Share final image and full chat thread.

Target image elements: backyard patio at sunset, circular fire pit with flames, lounge chairs with bright pillows, dining table and flowers, warm landscape lighting, pavers, gravel, planters, pink/purple sky, realistic upscale outdoor-living style.

Process: Generate baseline → compare to target → fix composition → fix lighting/color → fix object placement → use doodle mask for specific mismatches → save final image and chat share link → write reflection.

Minimum viable: 4 iterations + final image + chat link + short reflection. Strong: 6–8 iterations with notes on what each prompt improved.

Activity 2 starter prompt
Paste this into Gemini as your baseline image generation prompt.
Create a realistic upscale suburban backyard patio scene at sunset. The foreground should feature a circular concrete fire pit with visible flames. Around it, place modern dark lounge chairs with bright turquoise and red patterned pillows. 🧠 Weekly Big Ideas
›

1. AI interface design shapes trust

The same answer feels different in search, chat, voice, or video. Interface design affects how much users slow down and verify — and how much they anthropomorphize.

2. Convenience creates dependency risk

AI is useful because it removes friction. That same friction removal can create automation bias — users accept outputs they would otherwise verify.

3. Anthropomorphism is by design, not just error

AI products are built to feel conversational, warm, responsive, and personalized. Voice makes this effect stronger. Users aren't being irrational — the product is working as intended.

4. Sycophancy feels good but weakens judgment

The problem is not only wrong facts. AI agreement can feel like validation — users may stop pushing back on AI outputs because the AI stopped pushing back on them.

5. Prompting is responsible supervision

Prompting sets task, context, criteria, evidence, format, tone, and boundaries. The strongest AI user is not the one who gets output quickly — it's the one who can judge whether output deserves trust.

6. RAG helps, but doesn't solve trust

Source-grounded tools can still omit nuance, misframe evidence, or make weak connections sound stronger. Verification remains the user's responsibility regardless of grounding.

📓 Master Notes
›
Use one concept-first master note, not source-by-source notes. These headings organize Week 2 by idea, not by reading.

1. AI interfaces and search behavior

  • Core point: AI is moving from a tool you visit to an interface embedded across search, browsers, phones, image tools, and voice.
  • Key terms: AI browsers, AI Overviews, query fanout, AI search, publisher traffic, source verification.
  • Use for memo: Media ecosystem angle — who owns attribution when AI summarizes?
  • Use for Assignment 1: UX critique of how NotebookLM surfaces sources.

2. Overreliance and automation bias

  • Core point: AI reduces friction, but lower friction can weaken verification habits.
  • Key terms: overreliance, automation bias, errors of commission, errors of omission, miscalibrated trust.
  • Use for memo: Responsible use angle — the tool works but the user's judgment may not.
  • Use for Assignment 1: Does NotebookLM encourage or reduce overreliance?

3. Anthropomorphism and artificial intimacy

  • Core point: Users treat AI as more human than it is, especially in voice/companion contexts. This is partly by design.
  • Key terms: courtesy, reinforcement, roleplay, companionship, artificial intimacy.
  • Use for Activity 1: Spoken interaction reflection — did voice feel more human?
  • Use for memo: Voice/companionship angle.

4. Sycophancy and confirmation bias

  • Core point: AI can feel supportive while mirroring user assumptions instead of challenging them. RLHF training reinforces agreeableness.
  • Key terms: sycophancy, agreement bias, RLHF, confirmation bias.
  • Use for memo: Trust and AI literacy — feeling validated is not the same as being right.
  • Use for Assignment 1: Does NotebookLM push back on unsupported claims?

5. Prompting as responsible control

  • Core point: Prompting is not just productivity — it's how users communicate intent, constraints, and standards. Better prompts produce more verifiable outputs.
  • Key terms: prompt writing, prompt design, prompt engineering, reference text, examples, constraints, iteration, evaluation.
  • Use for Activity 2: Iterative image prompting with explicit criteria.
  • Use for Assignment 1: Test design — how well did your prompts stress-test NotebookLM?

6. RAG, source grounding, and hallucination reduction

  • Core point: RAG improves source grounding but does not automatically guarantee accurate interpretation. Human verification remains necessary.
  • Key terms: retrieval, grounding, citations, source quality, "I don't know," direct quotes, source restriction.
  • Use for Assignment 1: Central critique framework — NotebookLM is a RAG tool. How well does it ground claims?

Key tensions

  • Voice AI lowers friction but may raise anthropomorphism and lower verification.
  • Sycophancy feels like good UX but weakens critical thinking.
  • RAG reduces hallucination risk without eliminating the need for judgment.
  • Prompt literacy is a user responsibility, but product design shapes whether users bother.

📝 Reading Memo Guide
›
Memo goal: Synthesize Week 2, connect to your work/media use, include a substantive question, and include one outside resource. Do not summarize each reading one by one.

Strongest angle: Prompting as responsible supervision

Main claim: Prompting is not just a productivity technique — it is a form of responsible AI supervision that reconnects users to their own judgment.

Source connections: Prompting 101, RAG, journalist prompting, sycophancy, hallucination reduction, Week 2 lecture.

Professional connection: Creator platforms increasingly rely on AI-assisted workflows, so prompt literacy becomes part of platform responsibility.

Discussion question: Should prompt literacy be treated as a core form of media literacy?

Rough structure: (1) Open with voice chat observation. (2) State the concern: ease leads to overreliance, anthropomorphism, sycophancy. (3) Pivot: prompting and RAG offer ways to restore control, but only if users verify. (4) Connect to media/creator platforms. (5) End with question + outside resource.

Angle 2: Voice AI reduces critical distance

Main claim: Voice makes AI feel more human and helpful, but it can also increase anthropomorphism and reduce source-checking.

Source connections: Activity 1, NPR, anthropomorphism, sycophancy, Against Imaginary Friends.

Discussion question: Should voice AI interfaces include more reminders that users are interacting with a system rather than a person?

Angle 3: AI search changes the media trust chain

Main claim: AI summaries shift user attention away from original sources, creating new questions about attribution, context, and publisher value.

Source connections: AI search, RAG, source grounding, media economics.

Discussion question: If AI gives users answers without source visits, who becomes responsible for preserving context and attribution?
